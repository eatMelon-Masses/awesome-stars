# 项目收藏目录

本目录存放已收藏的 GitHub 项目的详细信息和 Issue 模板。

## 目录结构

每个项目包含两个文件：

1. **项目详细文档** (`项目名.md`)：包含项目的完整介绍、特性、技术栈、学习价值等详细信息
2. **Issue 模板** (`项目名-issue-template.txt`)：可直接复制的 Issue 内容模板，用于在 GitHub 仓库中创建 Issue

## 如何使用

### 查看项目详情

直接打开对应的 `.md` 文件即可查看项目的详细信息。

### 创建 GitHub Issue

1. 打开对应的 `-issue-template.txt` 文件
2. 复制文件中的全部内容
3. 在仓库中创建新 Issue
4. 将复制的内容粘贴到 Issue 正文中
5. 设置 Issue 标题，格式如：`项目名 - 简短描述`
6. 根据需要添加 GitHub Labels 进行分类
7. 提交 Issue

### 示例

以 clash-verge-rev 项目为例：

**Issue 标题**：
```
clash-verge-rev - 基于Tauri的Clash Meta GUI客户端
```

**Issue 内容**：
```
（复制 clash-verge-rev-issue-template.txt 的全部内容）
```

## 添加新项目

如需添加新项目到此目录，请：

1. 创建 `项目名.md` 文件，包含完整的项目信息
2. 创建 `项目名-issue-template.txt` 文件，包含简洁的 Issue 模板内容
3. 更新本 README 的"已收藏项目列表"部分
